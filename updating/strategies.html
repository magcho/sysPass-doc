方針¶

sysPassにはwebアプリによくある**自己アップデート機能はありません**。これはsysPass自体がGitなどのバージョン管理ツールを用いて管理・更新されることを想定しているためです。自己アップデート時にありがちな前のバージョンのファイルがアップデートで上書きされて消えてしまう心配がありません。そういった場合でもGitなどには強力なマージツールがあるので安心してsysPassのアップデートを行うことができます。

また、よく聞かれる質問ですが。Docker or Kubernetes を用いて構成している場合はどのようにアップデートを行うのか。これらについてもsshなどで接続してGitコマンドを用いてアップデートを行うこともできますがコンテナの再現性を損ねてしまいます。

昨今のコンテナ技術を考えるとsysPassイメージのタグを変更してアップデートする方法が一般的です。Docker Compose や Helm を用いることで簡単にこの操作を行うことができます。

Normal¶

アップデートの詳細はリリースノートを参照してください

Git¶

Git を用いてsysPassをインストールした場合は以下のようなコマンドを実行したはずです

$ git clone https://github.com/nuxsmin/sysPass.git

Git コマンドでクローンしたsysPassディレクトリ内で以下のコマンドを実行するとsysPassが最新のものにアップデートされます

$ git pull

注釈

Github上の “master” ブランチは常に最新のstableバージョンになっています

アップデートに伴いDBに変更が必要な場合はブラウザでsysPassにアクセスするとその旨が表示されます。具体的な手順についてはその表示に従ってください

警告

アップデートする前に”mysqldump” や “tar” などのコマンドを利用するなどして、sysPassディレクトリ全体とデータベース全体のバックアップをしてください

Docker¶

Docker Compose を用いて運用している場合のアップデートは簡単です。デフォルトのdocker-compose.ymlを利用している場合は起動時に常に最新のsysPassが使用されます。docker-compose.ymlを自分で書き換えている場合はtagを更新するだけでsysPassのアップデートができます。

version: '2'
services:
  app:
    container_name: syspass-app
    image: syspass/syspass:3.1.0 # Set this version tag to desired one
    restart: always
...

tagを変更した後に以下のコマンドを実行すると最新のsysPassがダウンロードされ、自動で起動します

docker-compose -p syspass -f docker-compose.yml up -d

アップデートに伴いDBに変更が必要な場合はブラウザでsysPassにアクセスするとその旨が表示されます。具体的な手順についてはその表示に従ってください

警告

アップデートする前にsysPassディレクトリ全体とデータベース全体のバックアップ バックアップ手順 をしてください
