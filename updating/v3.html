3.0 Version¶

このバージョンにアップデートするにはsysPassがv2.1である必要があります。2.1より前のバージョンをお使いの場合は一旦v2.1へアップデートの後v3.0にアップデートしてください

重要な変更¶

  • このバージョンではDBの構成が変更されています。必ず”mysqldump” コマンドなどを用いてDBをバックアップした後アップデート作業を行ってください

  • “config” ディレクトリの場所が “/app/config” に変更になりました

  • 依存パッケージをComposerを用いて管理するようになりました

アップデート手順¶

sysPassのアップデートの概略です、以下のような手順でアップデートを行ってください

  1. https://github.com/nuxsmin/sysPass/releases もしくは https://github.com/nuxsmin/sysPass.git から最新のsysPassを取得し、サーバに展開してください

  2. インストール時と同様にフォルダの所有者とパーミッション変更を行ってください

  3. ”…/config” 内にある “config.xml”, “key.pem”, “pubkey.pem” ファイルを “…/app/config” ディレクトリにコピーしてください

  4. sysPassのルートディレクトリにComposerをインストールしてください。詳しくは https://getcomposer.org/download/ を参照してください

  5. 以下のコマンドで依存パッケージをインストールしてください

$ php composer.phar install --no-dev

  1. インストール時と同様にフォルダの所有者とパーミッション変更を行ってください

  2. ブラウザでsysPassにアクセスしてください

  3. 必ず”mysqldump” コマンドなどを用いてDBのバックアップを取ってください

  4. “config/config.xml” ファイル内の “upgradeKey” に記載されているアップデートコードを、ブラウザの指示されたところに入力してください
