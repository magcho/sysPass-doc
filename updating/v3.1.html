3.1 Version¶

v3.0からのアップデートをすることを強く勧めます。お使いのsysPassがv3.0より低いバージョンの場合は段階を踏んでアップデートすることを推奨します。

アップデート情報のほかv2.1 から 3.0 Version へのアップデートの情報も参照してください

重要な変更¶

  • このバージョンはインストールされているextention(plugins)に影響するDBの変更があります、必ずDBのバックアップを取ってから アップデートと作業を行ってください

  • 依存パッケージをComposerを用いて管理するようになりました

アップデート手順¶

sysPassのアップデートの概略です、以下のような手順でアップデートを行ってください

  1. https://github.com/nuxsmin/sysPass/releases もしくは https://github.com/nuxsmin/sysPass.git から最新のsysPassを取得し、サーバに展開してください

  2. インストール時と同様にフォルダの所有者とパーミッション変更を行ってください

  3. “…/app/config” 内にある “config.xml”, “key.pem”, “pubkey.pem” ファイルをアップデート後の “…/app/config” ディレクトリにコピーしてください

  4. sysPassのルートディレクトリでComposerを用いて依存パッケージのインストールを行ってください。(Composerをインストールしていない場合は https://getcomposer.org/download/ を参照してインストールしてください)

  5. 以下のコマンドで依存パッケージをインストールしてください

$ php composer.phar install --no-dev

  1. インストール時と同様にフォルダの所有者とパーミッション変更を行ってください

  2. ブラウザでsysPassにアクセスし、表示されたアップグレードプロセスに従って処理を完了させてください

  3. アップデートする前に”mysqldump” コマンドを利用するなどして、データベース全体のバックアップをしてください

  4. “…/app/config/config.xml” ファイル内の “upgradeKey” に記載されているアップデートコードを、ブラウザの指示されたところに入力してください

  5. もしアップデートに失敗した時は “…/app/config/syspass.log” のエラーメッセージを参照してください

  6. アップデートが成功するとログイン画面にリダイレクトされます

警告

アップデートをやり直す際はそのまま再度アップデートプロセスを実行しないでください。最初のアップデートでデータベースの構造が変更されている可能性がありその場合再度実行してしまうと正常に処理を完了できません。アップデート前に取ったバックアップデータを用いてデータベースとsysPassを復元した後再度アップデートプロセスを実行してください。
