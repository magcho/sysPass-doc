CentOS 7.x へのインストール¶

動作用件¶

  • Web server [Apache/Nginx/Lighttpd] (SSLを有効化する際はSSLに対応するWeb serverを利用してください)

  • MariaDB >= 10.1

  • PHP >= 7.0 (7.1以上を推奨します)

  • PHP modules

    • mysqlnd

    • curl

    • json

    • gd

    • xml

    • mbstring

    • intl

    • readline

    • ldap (optional)

    • mcrypt (optional: 古いバージョンのsysPassで出力したXMKファイルをいインポートする際に必要です)

  • 最新版のsysPass (https://github.com/nuxsmin/sysPass/releases/latest)

  • リリースページのほかGitを利用して最新のsysPassを利用することもできます(https://github.com/nuxsmin/sysPass.git)

インストール¶

CentOS 7 に以下のパッケージをインストールしてください (https://www.softwarecollections.org/en/scls/rhscl/rh-php70/).

$ sudo yum -y install centos-release-scl.noarch
$ sudo yum -y install rh-php71 rh-php71-php rh-php71-php-fpm httpd rh-mariadb102 wget
$ sudo yum -y install rh-php71-php-gd rh-php71-php-intl rh-php71-php-json rh-php71-php-ldap rh-php71-php-mbstring rh-php71-php-mysqlnd rh-php71-php-opcache rh-php71-php-pdo rh-php71-php-xml rh-php71-php-zip

以下のコマンドでApache web serverとMariaDB serverが自動起動するように設定してください。

$ sudo systemctl enable httpd.service
$ sudo systemctl enable mariadb.service
$ sudo systemctl start httpd.service
$ sudo systemctl start mariadb.service

MariaDBの設定

$ sudo /usr/bin/mysql_secure_installation

以下のコマンドでファイヤーウォールでsysPassの利用に必要な通信を許可してください。

$ sudo firewall-cmd --permanent --zone=public --add-service=http
$ sudo firewall-cmd --permanent --zone=public --add-service=https
$ sudo firewall-cmd --reload

オプションとしてSSLを有効化することもできます。

SSLを利用することでより一層sysPassを安全に利用することができます。SSLの有効化手順については セキュリティー ページ、または以下を参照してください

  • sysPassをLAN内で利用している場合: https://doc.debian.org/configuration/Self-Signed_Certificate

  • sysPassをインターネットに公開している場合はLet’s Encryptの利用を検討してみてください( https://certbot.eff.org/)

ディレクトリとパーミッション¶

サーバーのドキュメントルートに sysPass用のディレクトリを作成してください。

$ sudo mkdir /var/www/html/syspass

リリースページからsysPassをダウンロードして先ほど作成したディレクトリの中に配置してください。

$ cd /var/www/html/syspass
$ sudo tar xzf syspass.tar.gz

リリースページからダウンロードしなくてもGitを用いて配置することもできます。

$ sudo git clone https://github.com/nuxsmin/sysPass.git  /var/www/html/syspass

以下のコマンドでいくつかのディレクトリの所有者とパーミッションを変更してください。

$ sudo chown apache -R /var/www/html/syspass
$ sudo chmod 750 /var/www/html/syspass/app/config /var/www/html/syspass/app/backup

SELinux¶

sysPassに構成ファイル(backup, cache, temp, etc)の書き込みを許可する必要があります。許可する方法は以下の２通りがあります。

注釈

どちらか一方の方法でsysPassの構成ファイルへの書き込みを許可してください。両方行う必要はありません

  • userの設定を変更する

$ sudo setsebool -P httpd_can_connect_ldap 1
$ sudo chcon -R -t httpd_sys_rw_content_t /var/www/html/syspass/app/{config,backup,cache,tmp}

  • SELinuxによる書き込み制限を無効化する。”/etc/sysconfig/selinux”ファイルを開き”SELINUX”の項目を”permissive”に変更してください。

依存パッケージをインストール¶

ComposerはPHP向けのパッケージマネージャーで依存パッケージのインストール・アップデートを簡単に行うために必要なものです。

Composerを利用しない場合はgithubのリリースページから “vendor.tar.gz” をダウンロードして配置することでsysPassを始めることができます。

注釈

Composerコマンドで依存パッケージをインストールするためにサーバーがインターネットに接続できるようになっている必要があります。

PHP Composerを利用する¶

sysPassのルートディレクトリへ移動し、Composerをインストールしてください、インストール方法はComposerのドキュメント（https://getcomposer.org/doc/faqs/how-to-install-composer-programmatically.md）を参照してください

以下のスクリプトを実行することでComposerをインストールすることもできます

#!/bin/sh
EXPECTED_SIGNATURE="$(wget -q -O - https://composer.github.io/installer.sig)"
php -r "copy('https://getcomposer.org/installer', 'composer-setup.php');"
ACTUAL_SIGNATURE="$(php -r "echo hash_file('sha384', 'composer-setup.php');")"

if [ "$EXPECTED_SIGNATURE" != "$ACTUAL_SIGNATURE" ]
then
    >&2 echo 'ERROR: Invalid installer signature'
    rm composer-setup.php
    exit 1
fi

php composer-setup.php --quiet
RESULT=$?
rm composer-setup.php
exit $RESULT

$ chmod +x install_composer.sh
$ ./install_composer.sh

Composerのインストールが終わったら、次はsysPassの依存パッケージをインストールします。以下のコマンドを実行してください

$ php composer.phar install --no-dev

環境設定¶

以上の操作が終わったらwebブラウザーでsysPassにアクセスしてインストールウィザードに従って環境設定をしてsysPassのインストールを完了してください

https://IP_OR_SERVER_ADDRESS/syspass/index.php

sysPassに関するもっと詳しい説明は Application をお読みください
