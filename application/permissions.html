Permissions(権限)¶

作成したユーザーのデフォルトの権限はアカウトの検索のみが許可されています。それ以外の操作を許可する場合はそれぞれの権限を付与してください。

sysPassの権限設定には29種類の設定項目があります

  • アカウント

    • Create - 新しいアカウントを作成することを許可します

    • View - アカウントの情報を閲覧することを許可します 1

    • View Password - アカウント情報と合わせて保存されているパスワードを閲覧することを許可します 1

    • Edit - アカウント情報を編集することを許可します 1

    • Edit Password - アカウント情報と合わせて保存されているパスワードを変更することを許可します 1

    • Delete - アカウントを削除することを許可します 1

    • Files - アカウント情報と合わせて保存されているファイルにアクセスすることを許可します

    • Share Link - 共有リンクを発行することを許可します

    • Private - 自分のみがアクセスできるプライベートなアカウトを作成することを許可します

    • Private for Group - 指定したグループに属するユーザーだけがアクセスできるプライベートアカウントを作成することを許可する

    • Permissions - 権限設定を閲覧したり、変更すること許可する 1

    • Global Search - プライベートアカウントを除く全てのアカウントを対象に検索をすることを許可する 2

  • Management

    • Users - 全てのユーザーを管理することを許可する 3

    • Groups - 全てのグループを管理することを許可する

    • Profiles - 全てのプロファイルを管理することを許可する

    • Categories - 全てのカテゴリーを管理することを許可する

    • Clients - 全てのクライアントを管理することを許可する

    • Custom Fields - 全てのカスタムフィールドを管理することを許可する

    • API Authorizations - 全てのAPIを管理することを許可する

    • Public Links - 全ての共有リンクについて管理することを許可する

    • Accounts - 全てのアカウントを管理することを許可する

    • Files - 全てのファイルを管理することを許可する

    • Tags - 全てのタグを管理することを許可します

  • Configuration

    • General - サイト・アカウント・Wiki・LDAP・メールに冠する全ての設定を変更することを許可します

    • Encryption - マスターパスワードに関する全ての設定を変更することを許可します

    • Backup - バックアップを作成することを許可します 4

    • Import - XML/CSVファイルをインポートすることを許可します

  • Others

    • Event Log - イベントログを閲覧することを許可します

ACL(Access Control List)¶

Users and Groups¶

  • ユーザープロファイルを設定することで権限を設定できます

  • ユーザーは次の場合に該当する時のみ、アカウント情報にアクセスできます

    • アカウントのオーナーである場合

    • 許可されたprimary groupに属している場合

    • 許可されたsecondary groupに属している場合

    • アカウントのsecondary userに設定されている場合

    • ユーザーのmian groupがアカウントのsecondary groupに設定されている場合

    • ユーザーのmian groupがアカウントのsecondary group accessオプションが有効化されている場合

  • 個人のプライベートアカウントには作成したユーザーとそのユーザーが許可したユーザーのみがアクセスできます

  • グループプライベートアカウントには許可されたグループがmain groupになっているユーザーのみがアクセスできます

  • アプリケーション管理者:: プライベートアカウントを除く全てのアカウント・全てのアプリケーションモジュール・アカウントへのフルアクセスをすることができます

  • アカウント管理車: プライベートアカウントを除く全てのアカウントにアクセスできます

API¶

APIにもユーザーと同じ権限が与えられているため、APIを用いてアクセスできる情報はブラウザで閲覧できる情報と同じです。

Notes

1(1,2,3,4,5,6)

そのユーザーがそのアカウントへのアクセス権限がある場合に限る

2

検索する権限は持っているが、そのアカウントへのアクセス権限を持っていないユーザーはそのアカウントにアクセスすることはできません。代わりに’アクセス権限のリクエスト’だけを行うことができます

3

‘アプリケーション管理者’以外が他のユーザーの情報を変更することはできません

4

生成したバックアップファイルをダウンロードできるのは’アプリケーション管理者’のみです
